# 相机 IO 联调清单

适用场景：PLC 通过硬件线直接触发海康相机采图，软件通过相机 SDK 取图检测，并在 NG 时通过相机输出线回传结果。

## 接线确认

1. 确认 PLC 触发线接到相机 `Line0+` 对应输入端，公共地和电平规格与相机型号一致。
2. 确认 NG 回传线接到相机 `Line1+` 对应输出端，PLC 输入端与相机输出类型匹配。
3. 核对相机说明书中的 IO 电平、是否需要外接电源、是否为光耦输入输出。

## 相机节点确认

1. 在 MVS 中确认 `TriggerMode = On`。
2. 在 MVS 中确认 `TriggerSource = Line0`。
3. 确认 `TriggerActivation` 与 PLC 实际输出边沿一致，常见为 `RisingEdge`。
4. 确认 `LineSelector = Line1` 时，`LineMode = Output`。
5. 确认 `LineSource` 可以配置到 `UserOutput0` 或相机型号实际支持的用户输出节点。

## 软件侧检查

1. 配方 `trigger_mode` 设为 `plc_external`。
2. 在配方编辑页“检测与IO”中确认 `相机输出线` 为 `Line1`，脉宽与延时符合现场节拍。
3. 主界面启动 `IO 监听` 后，状态应停留在“等待 Line0”。
4. PLC 发出一个有效脉冲后，软件应立即进入检测并刷新图像与结果。

## 结果回传检查

1. 制造一张稳定 OK 样本，确认不会误输出 Line1 脉冲。
2. 制造一张稳定 NG 样本，确认检测完成后 Line1 有脉冲输出。
3. 用示波器或 PLC 监控确认脉宽与设定值一致。
4. 若 PLC 收不到信号，优先检查 `LineSource`、输出类型、电平和公共地。

## 常见排查

1. 一直等待触发：先查 `TriggerSource` 是否仍为 `Line0`，再查 PLC 脉冲宽度是否过窄。
2. 相机能采图但软件不更新：检查上位机是否已启动 `IO 监听`，以及是否仍有其他程序占用相机。
3. NG 不回传：先查 `Line1` 是否支持输出，再查 `UserOutput0` 是否被型号支持。
4. 节拍波动大：先排除 PLC 触发抖动、曝光过长、网络丢包和磁盘保存阻塞。
