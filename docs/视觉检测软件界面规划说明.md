# 视觉检测软件界面规划说明

## 1. 文档目的

本文档用于规划视觉检测软件的一期界面方案，作为后续原型设计、PyQt 界面开发和交互实现的依据。

本文档覆盖以下界面：

1. 主界面
2. 配方编辑界面
3. 模板框选界面

界面规划目标如下：

1. 满足一期单工位单相机、多 ROI 检测的业务需求。
2. 支持 PLC 外部触发、检测结果显示、NG 标注和本地留档。
3. 支持直接在软件界面维护配方、模板和 ROI。
4. 保留后续扩展多模板和多工位的界面空间。
5. 界面视觉风格保持简约、高级、克制，避免过度装饰和控件堆叠。
6. 在全屏运行状态下，主流程相关内容应尽量一次完整显示，避免关键区域被折叠或滚动后才能看到。

## 2. 总体设计原则

### 2.1 设计原则

1. 检测主流程优先，编辑类功能次之。
2. 主界面应突出当前图像、检测结果和设备状态。
3. 编辑界面应突出配置效率和参数可读性。
4. ROI 框选界面应突出操作直观性和坐标精确性。
5. 界面布局应支持后续扩展而不推翻现有结构。
6. 控件间距和卡片尺寸应尽量紧凑，优先保证信息密度与扫描效率。
7. 配色应偏中性工业风，以浅底、低饱和强调色和少量状态色为主。
8. 非关键说明文字应弱化，关键状态、结果和报警应突出。

### 2.2 使用角色

一期主要面向以下角色：

1. 操作员：负责日常运行、查看结果、切换配方。
2. 工艺或调试人员：负责导入模板、框选 ROI、设置阈值。
3. 开发或设备调试人员：负责联调相机、PLC 和异常排查。

### 2.3 导航方式建议

建议使用以下导航结构：

1. 主界面作为软件默认首页。
2. 配方编辑界面以独立页面或独立对话框形式打开。
3. 模板框选界面从配方编辑界面进入。

建议避免将所有功能堆叠在同一窗口内，否则后期维护和使用都会很混乱。

### 2.4 字体与信息层级规范

为避免界面出现“全部都在喊重点”的问题，一期应建立统一的字体层级，不同信息按角色分层，而不是按控件类型随意放大加粗。

#### 字体家族建议

建议全局字体优先级如下：

1. Alibaba PuHuiTi 3.0
2. Alibaba PuHuiTi 2.0
3. Alibaba PuHuiTi
4. Microsoft YaHei UI
5. Microsoft YaHei
6. Segoe UI

建议所有中文界面文本统一使用同一主字族，不要在标题、表格、按钮之间混用不同中文字体。这样比单独调字号更能改善整体质感。

#### 字体层级建议

建议将一期界面文本控制为以下五档：

1. 一级标题：18 到 20 px，700 字重。用于主界面页眉标题、重要模块标题。
2. 二级标题：15 到 16 px，700 字重。用于卡片标题、图像区标题、日志区标题。
3. 状态与关键值：14 到 18 px，600 到 700 字重。用于顶部状态栏值、结果详情值、当前模板名。
4. 数字指标：22 到 40 px，700 到 800 字重。用于 OK 或 NG 总结果、良率、OK 数、NG 数、连续 NG 等核心统计指标。
5. 辅助文字：11 到 12 px，400 到 500 字重。用于字段名、表头说明、日志辅助信息、路径说明、页脚信息。

#### 使用约束建议

1. 同一屏内不要超过 3 种字号主档，避免界面碎。
2. 不要通过大量使用粗体来制造层级，优先通过字号、颜色明度和留白区分。
3. 数字指标需要比说明文字明显更强，但不能强到压过检测主结果。
4. 表格表头应比正文略强一档，正文保持稳定可读，避免把整张表做成高对比度粗体。
5. 页脚、路径、说明和次级状态信息应明显弱于主结果区与顶部运行状态。

## 3. 主界面规划

### 3.1 主界面定位

主界面用于现场运行与实时检测，是操作员最常使用的页面。其重点不是参数编辑，而是：

1. 观察系统状态
2. 查看图像结果
3. 查看每个 ROI 判定
4. 接收 NG 提示
5. 快速切换配方

### 3.2 主界面布局建议

建议主界面采用三段式结构：

1. 顶部状态栏
2. 中部主体区
3. 底部日志与快捷操作区

在视觉和布局上，建议进一步控制为“薄顶部、重主体、轻边栏”的紧凑比例：

1. 顶部仅保留高频状态信息与核心生产指标。
2. 操作按钮尽量并入右侧控制区，避免在图像区上方叠加过多工具栏。
3. 中部图像区与结果区应成为全屏时最主要的视觉区域。
4. 全屏下不应出现需要明显上下滚动才能看到检测结果、ROI 列表或关键控制按钮的情况。

建议的布局结构如下：

```text
+--------------------------------------------------------------------------------+
| 顶部状态栏: 工位 | 相机状态 | PLC状态 | 当前配方 | 当前模板 | 节拍 | 时间      |
+--------------------------------------------------------------------------------+
| 左侧图像显示区                          | 右侧结果与控制区                     |
|                                        |                                      |
| 实时图像 / 检测结果图                  | 配方选择                             |
| ROI 叠加显示                           | 检测总结果 OK/NG                     |
| NG 区域高亮                            | 各 ROI 结果列表                      |
|                                        | 统计信息                             |
|                                        | 手动测试按钮                         |
+--------------------------------------------------------------------------------+
| 底部日志区: 触发记录 | 检测结果 | 保存状态 | PLC输出状态 | 异常信息           |
+--------------------------------------------------------------------------------+
```

### 3.3 顶部状态栏设计

顶部状态栏建议固定显示以下信息：

1. 工位编号
2. 相机连接状态
3. PLC 通讯状态
4. 当前配方名称
5. 当前模板名称
6. 当前运行模式
7. 最近一次检测耗时
8. 当前系统时间

顶部状态栏建议采用紧凑卡片或标签式布局，不建议使用过高的大块面板。状态区高度应尽量受控，使更多垂直空间留给图像区和结果区。

#### 状态栏展示要求

1. 相机状态应直观区分已连接、未连接、异常。
2. PLC 状态应直观区分已连接、未连接、超时。
3. 当前配方和模板应始终可见。
4. 节拍超限时应有醒目提示。

#### 状态栏字体建议

1. 状态标题建议使用 11 px 左右、500 字重。
2. 状态值建议使用 14 到 15 px、600 字重。
3. 当前模式、PLC 状态、节拍等需要快速扫视的信息可使用更高对比色，但不建议再额外放大过多字号。

### 3.4 左侧图像显示区设计

左侧区域为主视觉区，建议占主界面宽度的 60% 到 70%。

#### 主要功能

1. 显示实时采集图像。
2. 显示检测完成后的结果图。
3. 叠加 ROI 矩形框。
4. 对 NG ROI 用红色边框或半透明遮罩高亮。
5. 支持显示 ROI 名称与分值。

#### 图像区交互建议

1. 正常运行时显示最近一次检测图像。
2. NG 时自动切换为结果标注图。
3. 支持缩放查看，但一期可先不支持复杂平移。
4. 支持鼠标悬停显示 ROI 名称、阈值、实际分值。

### 3.5 右侧结果与控制区设计

右侧区域建议从上到下划分为四个分区：

1. 配方控制区
2. 总结果区
3. ROI 结果列表区
4. 统计与操作区

为保证全屏时所有核心内容完整显示，建议右侧区域采用“上紧下松”的组织方式：

1. 配方控制区和运行控制区尽量做成紧凑按钮组。
2. 总结果区保持醒目，但不宜过高。
3. ROI 结果列表区占据右侧主要剩余空间。
4. 统计信息建议做成小型指标卡，不宜做大面积面板。

#### 3.5.1 配方控制区

建议包含以下控件：

1. 配方下拉框
2. 刷新配方按钮
3. 打开配方编辑按钮
4. 当前模板显示标签

#### 3.5.2 总结果区

建议使用醒目的大尺寸状态卡片显示：

1. OK 或 NG
2. 本次检测耗时
3. 触发时间
4. NG 数量

颜色建议：

1. OK 使用绿色
2. NG 使用红色
3. 未触发或等待状态使用灰色或蓝色

排版建议：

1. OK 或 NG 主结果建议使用 32 到 40 px、800 字重，作为右侧最强视觉锚点。
2. 耗时、触发时间、NG 数量等辅助结果建议使用 11 px 标题加 16 到 18 px 数值的双层结构。
3. 同一个结果卡中不应让所有数字一样大，否则主次不清。

#### 3.5.3 ROI 结果列表区

建议采用表格形式，列出每个 ROI 的检测结果。

建议表头如下：

1. 序号
2. ROI 名称
3. 阈值
4. 实际分值
5. 状态
6. 备注

交互建议：

1. 点击某一行时，在左侧图像区高亮对应 ROI。
2. NG 行应自动高亮为红色。
3. 分值低于阈值时显示差值或异常说明。

排版建议：

1. 表头建议使用 11 到 12 px、600 到 700 字重。
2. 表格正文建议使用 10 到 11 px、400 到 500 字重。
3. 分值列、状态列和序号列建议居中，状态列允许比普通正文略强一档。
4. 不建议把整行文字全部加粗，重点只落在状态和异常行颜色上。

#### 3.5.4 统计与操作区

建议显示：

1. 今日 OK 数
2. 今日 NG 数
3. 连续 NG 次数
4. 最近一次保存路径

统计卡片建议控制为单行或双行紧凑布局，避免在右侧形成过长的纵向堆叠。

建议操作按钮：

1. 手动测试
2. 保存当前图像
3. 查看记录
4. 清除报警

### 3.6 底部日志区设计

底部日志区用于显示最近运行事件，建议采用滚动文本框或列表。

如果主界面高度有限，一期可先将底部日志区弱化为状态栏消息或可折叠日志区，以保证图像、总结果和 ROI 列表优先完整显示。

应显示以下信息：

1. PLC 触发到达
2. 图像采集成功或失败
3. 检测完成
4. 图片保存完成
5. PLC NG 输出完成
6. 异常信息

### 3.7 主界面关键状态流转

主界面应至少支持以下状态：

1. 初始化中
2. 待机中
3. 等待 PLC 触发
4. 正在采图
5. 正在检测
6. 检测完成 OK
7. 检测完成 NG
8. 异常状态

建议在主界面用明显的状态标签统一展示当前状态。

## 4. 配方编辑界面规划

### 4.1 配方编辑界面定位

配方编辑界面面向工艺或调试人员，负责维护与检测相关的全部核心参数。这个页面是配置中心，不直接承担实时检测展示任务。

### 4.2 配方编辑界面布局建议

建议采用左右分栏布局：

1. 左侧为配方和模板树
2. 右侧为属性编辑区

布局结构建议如下：

```text
+--------------------------------------------------------------------------------+
| 顶部工具栏: 新建配方 | 保存 | 另存为 | 删除 | 导入模板 | 打开框选界面        |
+--------------------------------------------------------------------------------+
| 左侧导航树                     | 右侧属性编辑区                            |
|                                |                                           |
| 配方列表                       | 配方基本信息页                            |
|   └ 模板A                      | 模板信息页                                |
|      └ ROI 1                   | ROI 列表与阈值页                          |
|      └ ROI 2                   | PLC 参数页                                |
|   └ 模板B                      | 图像保存参数页                            |
|                                | 运行参数页                                |
+--------------------------------------------------------------------------------+
| 底部状态区: 配方保存状态 | 更新时间 | 当前文件路径                             |
+--------------------------------------------------------------------------------+
```

### 4.3 左侧导航树设计

左侧建议使用树形结构展示配方内容：

1. 根节点为当前配方
2. 第二层为模板列表
3. 第三层为模板下的 ROI 列表

这样做的好处：

1. 结构清晰
2. 后续扩展多个模板时无需改布局
3. 点击任意节点都可以在右侧显示对应属性

### 4.4 顶部工具栏设计

建议包含以下操作：

1. 新建配方
2. 打开配方
3. 保存
4. 另存为
5. 删除配方
6. 新增模板
7. 删除模板
8. 打开模板框选界面
9. 导入模板图
10. 导出配方

### 4.5 右侧属性编辑区设计

右侧属性编辑区建议采用选项卡或分组面板形式。

建议包括以下分区：

1. 配方基本信息
2. 模板信息
3. ROI 列表与阈值
4. PLC 参数
5. 图像保存参数
6. 运行参数

#### 4.5.1 配方基本信息分区

建议字段如下：

1. 配方名称
2. 配方编号
3. 产品名称
4. 产品型号
5. 工位编号
6. 相机编号
7. 描述
8. 是否启用

#### 4.5.2 模板信息分区

建议字段如下：

1. 模板名称
2. 模板编号
3. 模板图片路径
4. 图像尺寸
5. 是否默认模板
6. 是否启用
7. 模板说明

建议这里同时显示模板缩略图。

#### 4.5.3 ROI 列表与阈值分区

建议使用表格管理 ROI，字段包括：

1. 序号
2. 名称
3. x
4. y
5. 宽
6. 高
7. 阈值
8. 算法
9. 启用状态

表格上方建议放置按钮：

1. 新增 ROI
2. 删除 ROI
3. 编辑 ROI
4. 打开框选界面

#### 4.5.4 PLC 参数分区

建议字段如下：

1. PLC 启用状态
2. 触发来源
3. 通讯协议
4. 地址或端口配置
5. NG 输出通道
6. 输出延时
7. 脉冲宽度
8. 超时时间

#### 4.5.5 图像保存参数分区

建议字段如下：

1. 数据根目录
2. 是否保存原图
3. 是否保存结果图
4. 是否仅保存 NG 图
5. 是否生成 JSON 记录
6. 是否生成 CSV 汇总
7. 文件保留天数

#### 4.5.6 运行参数分区

建议字段如下：

1. 目标节拍
2. 检测超时
3. 采图失败重试次数
4. 是否允许手动测试

### 4.6 配方编辑界面关键交互

1. 选择模板节点时，右侧自动切换到模板信息和 ROI 配置。
2. 选择 ROI 节点时，右侧可直接编辑 ROI 参数。
3. 导入模板图后，应自动刷新模板预览和图像尺寸。
4. 点击打开框选界面时，应把当前模板图和 ROI 列表传入框选页面。
5. 保存时应校验必填项和坐标合法性。

### 4.7 配方编辑界面校验规则

至少应校验以下内容：

1. 配方名称不能为空。
2. 相机编号不能为空。
3. 至少存在一个启用模板。
4. 模板图路径必须有效。
5. ROI 不能越界。
6. ROI 阈值必须在合法范围内。

## 5. 模板框选界面规划

### 5.1 模板框选界面定位

模板框选界面用于在模板图像上直接创建和编辑 ROI，是工艺设置效率的关键界面。

### 5.2 模板框选界面布局建议

建议采用左中右三栏布局：

1. 左侧为工具栏和模板信息
2. 中间为大图框选画布
3. 右侧为 ROI 属性面板

布局结构建议如下：

```text
+--------------------------------------------------------------------------------+
| 顶部工具栏: 保存 | 撤销 | 重做 | 适应窗口 | 1:1显示 | 新建ROI | 删除ROI       |
+--------------------------------------------------------------------------------+
| 左侧工具区         | 中间框选画布                           | 右侧属性区       |
|                    |                                        |                  |
| 模板名称           | 模板图大图显示                         | ROI名称          |
| 图像尺寸           | 鼠标拖拽框选                           | 坐标x y          |
| 缩放比例           | 显示已有 ROI                           | 宽高             |
| ROI 列表简表       | 当前 ROI 高亮                          | 阈值             |
|                    |                                        | 算法             |
|                    |                                        | 启用状态         |
+--------------------------------------------------------------------------------+
| 底部状态栏: 鼠标坐标 | 当前ROI尺寸 | 保存状态 | 提示信息                       |
+--------------------------------------------------------------------------------+
```

### 5.3 中间框选画布设计

框选画布是本界面的核心。

#### 主要功能

1. 显示模板原图。
2. 支持鼠标按下拖拽生成矩形 ROI。
3. 支持点击选中已有 ROI。
4. 支持拖动 ROI 改变位置。
5. 支持拖动控制点调整 ROI 尺寸。
6. 支持同时显示多个 ROI。
7. 支持当前选中 ROI 高亮。

#### 建议交互细节

1. 新建 ROI 时，默认生成名称，如 ROI_01。
2. 鼠标移动时底部状态栏显示图像坐标。
3. 选中 ROI 时右侧属性区同步刷新。
4. 删除 ROI 前给出确认提示。
5. ROI 越界时禁止保存并高亮提示。

### 5.4 左侧工具区设计

建议显示以下内容：

1. 模板名称
2. 模板图片路径
3. 图像尺寸
4. 当前缩放比例
5. ROI 简要列表

ROI 列表建议显示：

1. 序号
2. 名称
3. 阈值
4. 启用状态

点击左侧 ROI 列表中的项时，应同步选中画布中的对应 ROI。

### 5.5 右侧 ROI 属性区设计

右侧属性区用于精确编辑当前 ROI，建议包含：

1. ROI 名称
2. x 坐标
3. y 坐标
4. 宽度
5. 高度
6. 相似度阈值
7. 检测算法
8. 启用状态
9. 备注

建议支持两种修改方式：

1. 通过画布拖拽调整
2. 通过右侧输入框精确输入

### 5.6 顶部工具栏设计

建议按钮如下：

1. 保存
2. 取消
3. 撤销
4. 重做
5. 适应窗口
6. 1:1 显示
7. 放大
8. 缩小
9. 新建 ROI
10. 删除 ROI

### 5.7 模板框选界面关键交互

1. 打开界面时自动加载当前模板和已有 ROI。
2. 新建 ROI 后立即进入可编辑状态。
3. 修改坐标或尺寸时，画布和属性区同步刷新。
4. 阈值修改后实时保存到当前 ROI 临时状态。
5. 点击保存后统一回写到配方编辑界面。

### 5.8 模板框选界面校验规则

1. ROI 宽高必须大于最小值。
2. ROI 不能超出图像边界。
3. ROI 名称不能为空。
4. 阈值必须在 0 到 1 范围内。
5. 至少保留一个启用 ROI 才允许完成当前模板配置。

## 6. 三个界面之间的关系

三个界面之间建议按以下逻辑连接：

1. 主界面负责运行和查看结果。
2. 配方编辑界面负责维护配方、模板和参数。
3. 模板框选界面负责可视化创建和编辑 ROI。

建议交互链路如下：

1. 在主界面点击配方管理，打开配方编辑界面。
2. 在配方编辑界面选择模板后，点击框选按钮，打开模板框选界面。
3. 在模板框选界面完成 ROI 编辑后保存，返回配方编辑界面。
4. 在配方编辑界面保存配方后，主界面刷新当前配方和模板信息。

## 8. 开发落地建议

### 8.1 PyQt 页面拆分建议

建议按以下方式拆分窗口或组件：

1. MainWindow：主界面
2. RecipeEditorDialog 或 RecipeEditorWindow：配方编辑界面
3. RoiEditorDialog 或 TemplateCanvasWindow：模板框选界面
4. RoiResultTableWidget：ROI 结果表格组件
5. ImageCanvasWidget：图像显示与 ROI 叠加组件

### 8.2 组件职责建议

1. ImageCanvasWidget 负责图像显示、ROI 绘制、选中和拖拽。
2. RoiResultTableWidget 负责显示 ROI 分值和状态。
3. RecipeEditorWindow 负责配方对象的读取、编辑和保存。
4. TemplateCanvasWindow 负责 ROI 的图形化编辑。

### 8.3 状态同步建议

界面间建议通过统一的数据对象同步，而不要用大量零散控件字段直接互传。建议流程为：

1. 读取配方 JSON 到内存对象。
2. 各界面编辑同一个数据对象或其副本。
3. 点击保存时统一序列化回 JSON。

## 10. 结论

本界面规划方案以主界面为运行核心，以配方编辑界面为配置核心，以模板框选界面为 ROI 维护核心，能够满足当前项目一期的业务需求，并可较平滑扩展到多模板和多工位场景。后续进入开发阶段时，建议优先先做主界面和图像画布组件，再完成 ROI 编辑和配方持久化。
