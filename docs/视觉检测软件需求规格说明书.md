# 视觉检测软件需求规格说明书

## 1. 文档信息

- 文档名称：视觉检测软件需求规格说明书
- 文档版本：V1.0
- 编写日期：2026-05-18
- 适用阶段：项目立项、方案评审、开发实施、测试验收
- 项目性质：基于海康机器人工业相机 SDK 的配方化视觉检测软件

## 2. 项目背景

当前现场需要基于海康机器人工业相机完成产品外观或区域一致性检测。不同产品对应不同检测规则，检测动作由 PLC 外部触发，系统在接收到触发后完成拍照、图像比对、结果判定、结果保存及异常输出。

现有工作区已包含海康工业相机 Python SDK 示例代码，可用于相机设备枚举、打开关闭、取流、参数设置和图像保存。项目目标不是简单改造示例界面，而是在现有 SDK 能力基础上开发一套可用于现场部署的视觉检测软件。

## 3. 建设目标

系统应实现以下目标：

1. 以检测配方管理不同产品的检测逻辑。
2. 由 PLC 发出外部触发信号，驱动相机完成图像采集。
3. 支持在软件界面直接框选多个 ROI 检测区域。
4. 将采集图像与模板图像的对应 ROI 区域进行相似度比对。
5. 根据用户可配置阈值输出 OK 或 NG 判定结果。
6. 在界面中显示实时检测结果，并对 NG 区域进行高亮标注。
7. 在 NG 场景下通过 PLC 输出 NG 信号。
8. 保存每次检测的图片和结果记录至本地文件。
9. 软件架构支持后续扩展为一个配方对应多个模板，以及支持多个工位相机。

## 4. 适用范围

本需求文档适用于以下场景：

1. 单工位单相机的第一期项目建设。
2. 基于固定安装位置和相对稳定光照环境的模板比对检测。
3. 由 PLC 统一管理触发与结果联动的自动化产线场景。

本期不包含以下内容：

1. 深度学习模型训练与部署。
2. 云端数据库或远程服务平台建设。
3. MES、ERP 等上层系统对接。
4. 跨产线集中式调度平台。

## 5. 术语定义

- 配方：针对某一产品建立的一组检测规则及参数集合。
- 模板：用于比对的标准图像，可属于某个配方。
- ROI：检测区域，即在模板图或待测图像中参与比对的矩形区域。
- OK：检测通过。
- NG：检测不通过。
- PLC：可编程逻辑控制器，负责外部触发输入及 NG 信号交互。
- 外部触发：由 PLC 发出的采图指令信号。

## 6. 总体业务流程

系统总体业务流程如下：

1. 软件启动并完成初始化。
2. 系统连接指定工位相机并加载当前检测配方。
3. PLC 发出触发信号。
4. 系统接收触发并驱动相机采集单帧图像。
5. 系统对采集图像进行预处理。
6. 按照当前模板配置的多个 ROI 区域执行相似度比对。
7. 输出每个 ROI 的检测分值和单项结果。
8. 汇总生成最终 OK 或 NG 判定。
9. 若结果为 NG，则在界面高亮 NG 区域，并通过 PLC 输出 NG 信号。
10. 保存本次检测图片和本地结果记录。
11. 系统重新进入等待 PLC 触发状态。

## 7. 总体功能范围

本系统应包括以下功能模块：

1. 相机控制模块
2. PLC 通讯与触发联动模块
3. 配方管理模块
4. 模板管理模块
5. ROI 编辑模块
6. 图像预处理与相似度计算模块
7. 结果判定模块
8. 检测结果展示模块
9. 图片与记录保存模块
10. 系统配置与日志模块

## 8. 功能需求

### 8.1 相机控制模块

#### 8.1.1 功能目标

负责工业相机设备的枚举、连接、参数设置、触发模式切换、图像采集及异常处理。

#### 8.1.2 功能要求

1. 支持枚举当前可用海康工业相机设备。
2. 支持选择指定相机并建立连接。
3. 支持设置曝光、增益、帧率等基础参数。
4. 支持切换到外部触发模式。
5. 支持接收一次外部触发后采集单帧图像。
6. 支持检测相机断开、打开失败、采图失败等异常状态。
7. 支持软件启动后自动连接默认相机。
8. 支持在界面显示当前相机连接状态。

#### 8.1.3 输入

1. 相机选择信息
2. 相机参数配置
3. 外部触发信号

#### 8.1.4 输出

1. 当前采集图像
2. 相机状态信息
3. 错误码与异常日志

### 8.2 PLC 通讯与触发联动模块

#### 8.2.1 功能目标

负责接收 PLC 触发信号，并在检测结果为 NG 时输出 NG 信号。

#### 8.2.2 功能要求

1. 支持接收 PLC 发出的外部触发信号。
2. 在收到有效触发后启动单次采图和检测流程。
3. 检测结果为 NG 时，向 PLC 输出 NG 信号。
4. 支持配置 PLC 通讯参数或输出参数。
5. 支持配置 NG 信号保持时间、延时输出时间和复位方式。
6. 支持触发超时、通讯失败、输出失败等异常记录。
7. 支持在界面显示 PLC 通讯状态。

#### 8.2.3 说明

本阶段仅明确 PLC 是输入触发和输出 NG 的上位接口。具体通讯协议、物理接口和实现方式在技术设计阶段进一步确认。

### 8.3 配方管理模块

#### 8.3.1 功能目标

负责不同产品检测规则的创建、存储、编辑和切换。

#### 8.3.2 功能要求

1. 支持新建配方。
2. 支持编辑配方基本信息。
3. 支持删除配方。
4. 支持复制已有配方生成新配方。
5. 支持加载指定配方。
6. 支持保存配方到本地文件。
7. 支持显示当前配方版本或修改时间。
8. 支持为配方绑定相机、模板和默认阈值参数。

#### 8.3.3 配方基本信息建议字段

1. 配方编号
2. 配方名称
3. 产品型号
4. 工位编号
5. 相机编号
6. 模板列表
7. ROI 列表
8. 图片保存策略
9. 结果记录策略
10. PLC 输出参数
11. 创建时间
12. 修改时间

### 8.4 模板管理模块

#### 8.4.1 功能目标

负责模板图像的导入、加载、维护与切换。

#### 8.4.2 功能要求

1. 支持导入模板图。
2. 支持一个配方关联一个或多个模板。
3. 支持显示模板预览图。
4. 支持编辑模板名称和说明。
5. 支持设置模板启用状态。
6. 支持删除模板。
7. 支持模板与 ROI 的关联管理。

#### 8.4.3 设计约束

第一期可按单模板完成业务闭环，但数据结构必须按多模板扩展方式设计。

### 8.5 ROI 编辑模块

#### 8.5.1 功能目标

支持用户直接在软件界面上对模板图进行多区域框选和参数配置。

#### 8.5.2 功能要求

1. 支持在模板图像上直接框选多个矩形 ROI。
2. 支持拖拽调整 ROI 位置和大小。
3. 支持删除 ROI。
4. 支持为每个 ROI 命名。
5. 支持为每个 ROI 设置单独相似度阈值。
6. 支持设置 ROI 启用或禁用状态。
7. 支持保存 ROI 配置到本地文件。
8. 支持重新加载 ROI 配置。
9. 支持在检测界面叠加显示 ROI 框。

#### 8.5.3 ROI 数据项建议

1. ROI 编号
2. ROI 名称
3. 左上角坐标 x
4. 左上角坐标 y
5. 宽度 w
6. 高度 h
7. 相似度阈值
8. 启用状态
9. 备注

### 8.6 图像预处理与相似度计算模块

#### 8.6.1 功能目标

负责对采集图像进行标准化处理，并与模板区域进行相似度计算。

#### 8.6.2 功能要求

1. 支持对采集图像进行灰度化处理。
2. 支持对 ROI 图像进行尺寸统一。
3. 支持基础降噪处理。
4. 支持按 ROI 一一执行相似度计算。
5. 支持输出每个 ROI 的相似度分值。
6. 支持相似度算法参数配置。

#### 8.6.3 业务约束

1. 现场产品位置基本固定。
2. 现场光照相对稳定。
3. 一张图像需支持多个 ROI 并行或串行比对。

#### 8.6.4 算法建议

第一期建议优先采用基于固定 ROI 的结构相似度或模板匹配方案，以满足开发周期和现场节拍要求。后续若存在位置偏移问题，可扩展局部搜索、定位补偿或图像配准能力。

### 8.7 结果判定模块

#### 8.7.1 功能目标

负责根据各 ROI 的相似度结果生成最终检测结论。

#### 8.7.2 判定规则

1. 每个启用 ROI 独立计算相似度。
2. 若 ROI 相似度大于或等于对应阈值，则该 ROI 判定为通过。
3. 若所有启用 ROI 均通过，则最终结果为 OK。
4. 若任一启用 ROI 未通过，则最终结果为 NG。

#### 8.7.3 输出要求

1. 输出每个 ROI 的相似度数值。
2. 输出每个 ROI 的 OK 或 NG 状态。
3. 输出最终 OK 或 NG 结果。
4. 输出 NG 原因明细。

### 8.8 检测结果展示模块

#### 8.8.1 功能目标

负责向用户直观展示当前检测状态、每个 ROI 的结果以及整体判定结果。

#### 8.8.2 功能要求

1. 显示当前工位和当前配方。
2. 显示当前模板名称。
3. 显示当前采集图像。
4. 叠加显示所有 ROI 区域框。
5. 显示每个 ROI 的实时相似度。
6. 显示每个 ROI 的单项判定结果。
7. 显示最终 OK 或 NG。
8. 显示本次检测耗时。
9. NG 时高亮不合格 ROI。
10. NG 时显示不合格区域名称和相似度差值。

### 8.9 图片与记录保存模块

#### 8.9.1 功能目标

负责将每次检测的图像和结果记录保存到本地文件系统。

#### 8.9.2 功能要求

1. 支持保存原始采集图像。
2. 支持保存带标注的结果图像。
3. 支持按日期或配方自动创建目录。
4. 支持配置保存全部图片或仅保存 NG 图片。
5. 支持保存每次检测的结果记录。
6. 支持记录本次使用的配方、模板、ROI 分值和最终结果。
7. 支持记录 PLC 输出状态和异常信息。

#### 8.9.3 结果记录建议字段

1. 记录编号
2. 检测时间
3. 工位编号
4. 配方名称
5. 模板名称
6. 原图路径
7. 结果图路径
8. 各 ROI 相似度
9. 各 ROI 判定结果
10. 最终结果
11. 检测耗时
12. PLC 输出结果
13. 异常信息

#### 8.9.4 存储方式

结果记录仅保存本地文件，不接入数据库。记录文件格式建议使用 CSV 或 JSON。

### 8.10 系统配置与日志模块

#### 8.10.1 功能目标

负责系统级参数配置、异常日志和运行信息记录。

#### 8.10.2 功能要求

1. 支持保存系统初始化配置。
2. 支持保存相机默认参数。
3. 支持保存 PLC 默认通讯参数。
4. 支持保存日志输出级别。
5. 支持记录启动、停止、触发、采图、检测、保存、输出等关键事件。
6. 支持记录错误码和异常堆栈信息。

## 9. 非功能需求

### 9.1 性能要求

1. 单次检测总节拍应小于 1 秒。
2. 系统应在 PLC 触发后尽快完成采图、检测和结果输出。
3. 图像保存与记录保存不应明显阻塞主检测流程。
4. 在单工位单相机场景下，系统应满足连续稳定运行。

### 9.2 稳定性要求

1. 系统应支持长时间连续运行。
2. 相机异常断开后应支持告警及恢复策略。
3. PLC 通讯异常时应提示并记录日志。
4. 本地文件保存失败时应提示并记录异常。

### 9.3 易用性要求

1. 用户可通过界面完成配方切换。
2. 用户可通过界面完成模板导入和 ROI 框选。
3. 用户可直接查看每个 ROI 的检测分值和最终结果。
4. 界面应清晰区分 OK 和 NG 状态。

### 9.4 可维护性要求

1. 软件应采用模块化结构，避免将全部业务逻辑写入单一脚本。
2. 配方、模板和 ROI 配置应与代码解耦。
3. 后续新增产品时应通过新增配方完成，不应修改核心程序逻辑。

### 9.5 可扩展性要求

1. 软件架构应支持一个配方对应多个模板。
2. 软件架构应支持扩展为多个工位相机。
3. 检测算法应支持后续新增定位补偿或其他相似度方法。
4. 本地存储结构应支持后续扩展更多统计字段。

## 11. 外部接口需求

### 11.1 相机接口

1. 使用海康机器人工业相机 SDK。
2. 通过现有 Python SDK 示例代码进行二次开发。
3. 需支持外部触发采图模式。

### 11.2 PLC 接口

1. PLC 向系统发送触发信号。
2. 系统向 PLC 输出 NG 信号。
3. 具体物理接口和协议在技术设计阶段确定。

### 11.3 文件接口

1. 配方文件保存在本地。
2. 模板图片保存在本地。
3. 检测图片保存在本地。
4. 检测记录文件保存在本地。

## 13. 项目一期范围定义

一期项目建议交付范围如下：

1. 单工位单相机。
2. PLC 外部触发采图。
3. 单配方单模板检测闭环。
4. 单张图支持多个 ROI。
5. ROI 独立阈值配置。
6. 相似度比对并输出 OK 或 NG。
7. NG 区域界面高亮显示。
8. 检测结果本地图片和记录保存。
9. NG 时输出 PLC 联动信号。
10. 检测节拍小于 1 秒。

## 14. 二期扩展方向

二期建议扩展方向如下：

1. 一个配方支持多个模板自动匹配。
2. 多工位多相机并行检测。
3. 更复杂的定位和纠偏能力。
4. 统计报表与产量分析。
5. 用户权限和操作审计。
6. 与 MES 或其他上位系统对接。

## 15. 验收标准建议

项目验收建议至少满足以下标准：

1. 软件可正常连接海康工业相机并稳定采图。
2. 软件可接收 PLC 触发信号并完成单次检测。
3. 软件可在界面完成模板导入和多 ROI 框选。
4. 软件可对多个 ROI 输出相似度和最终判定。
5. NG 时界面可明确标注不合格区域。
6. NG 时可向 PLC 输出 NG 信号。
7. 每次检测可保存本地图片与记录文件。
8. 单次检测节拍满足小于 1 秒要求。
9. 软件在连续运行场景下无明显异常崩溃或卡死。

## 16. 风险与说明

1. 若现场产品位置波动超出预期，固定 ROI 比对方案可能出现误判，需要增加定位补偿能力。
2. 若光照稳定性不足，单纯相似度算法可能受影响，需要增加光照归一化处理。
3. PLC 接口协议未最终确认前，通讯联动实现需保留技术选型空间。
4. 若后续直接扩展到多工位，建议在一期开发中即完成模块化和线程模型设计。

## 17. 结论

本项目一期应围绕“配方化管理、PLC 触发、多 ROI 比对、NG 标注、NG 输出、本地留档、节拍小于 1 秒”这一主线展开。需求边界已具备进入系统设计、数据结构设计、界面设计和开发排期的条件。
